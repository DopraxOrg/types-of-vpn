Four tiers of visibility.

A taxonomy of VPN protocols by detection resistance
Not all VPNs
look alike — on the wire
Axis of measurement
Detectability
obvious invisible

Two servers running "a VPN" can behave completely differently on real-world networks. The difference is almost always the protocol — specifically, how easy it is to distinguish from ordinary web traffic. The spectrum runs from protocols that announce themselves with a simple packet signature, to mimicry layers that serve a real website to anyone who asks.

01/04
Resistance
Low
Easy to identify

Clear signature, plain sight.

The default set. Fast, well-understood, widely deployed — and broadcasting what they are to anyone inspecting the wire. Perfectly adequate on networks that don't filter by protocol class, which is most of them.

A simple packet signature — or, in some cases, the port number alone.

Resistance

TLS-wrapped, still fingerprintable

Dressed as HTTPS. Almost.

These protocols ride on TLS, so at a glance they look like ordinary web traffic. The giveaway is in the details: handshake ordering, cipher preferences, and extension quirks that differ from what a real browser produces.

TLS handshake fingerprinting — JA3/JA4-class analysis.

03/04
Resistance
High
Camouflaged as web traffic

Convincing, but probe-able.

Designed to blend in. Simple DPI rules won't catch them. More sophisticated approaches still can — by analyzing the statistical entropy of packet headers, or by "active probing," where the inspector itself connects to the server to see whether it behaves like a real web host.

Entropy analysis of headers, or active probing of the server.

04/04
Resistance
Maximum
Indistinguishable from web traffic

Mimicry, weaponized.

Not VPN protocols themselves — mimicry layers you bolt onto an existing tunnel via the Pluggable Transports interface. Cloak goes further: configure a "donor" website, and anyone who probes your server gets served real content from that site. The tunnel is invisible beneath a working web host.

In practice — only sloppy configuration (bad certs, port choices).

Operating Note I
A single protocol is a single point of failure. Running two — one fast, one stealthy — costs little and buys a lot.
Operating Note II
Tier is relative. A Tier 1 protocol is entirely adequate on a network that doesn't inspect traffic.
Operating Note III
The stack is separable: pick a tunnel, pick a camouflage layer, swap either without touching the other.
